# Byeoldori Project — Documentation Repository

본 저장소는 별도리 프로젝트의 개발 관련 문서를 관리하는 저장소 입니다.

**기획 → 요구사항 → 설계 → 개발 → 테스트 → 결과보고** 에 이르는 전체 문서 흐름을 포함합니다.

---

## 📄 문서 목록

### 01. 프로젝트 아이디어 개요서

초기 아이디어, 문제 정의, 필요성, 대상 사용자 분석을 기반으로
별도리 프로젝트의 기획 배경과 개발 필요성을 명확히 제시한 문서입니다.
국내 천문교육 환경의 문제점을 근거로 제시하며, 앱 개발의 목적을 설명합니다.

[프로젝트 아이디어 개요서](./Byeoldori_Doc_01_IdeaOverview_v6_2024-12-06.pdf)


### **02. 요구사항 정의서**

시스템이 반드시 제공해야 할 기능·비기능 요구사항,
유즈케이스, 인터페이스 구조, 시나리오 흐름을 포함한
프로젝트의 핵심 기준 문서입니다. 앱 개발의 모든 기준이 되는 문서입니다.

[요구사항 정의서](./Byeoldori_Doc_02_RequirementSpec_v4_2025-11-12.pdf)


### **03. 프로젝트 결과보고서**

프로젝트 전체 수행 과정 요약, 개발 환경, 기술 분석, 팀 역할, 선행기술 조사,
최종 산출물 평가 등을 포함한 최종 완성 보고 문서입니다.
별도리 앱이 어떤 가치와 결과를 냈는지 종합적으로 정리합니다.

[프로젝트 결과보고서](./Byeoldori_Doc_03_FinalReport_v3_2025-11-12.pdf)


### **04. 프로젝트 관리 계획서**

개발 일정(빌드별 목표), 팀 구성, 개발 프로세스(Agile), 품질 관리 방법,
고객/이해관계자 관리 등을 담은 프로젝트 운용 지침 문서입니다.
전체 개발 과정에서 “어떻게 개발을 운영했는가”를 정의합니다.

[프로젝트 관리 계획서](./Byeoldori_Doc_04_ProjectManagement_v2_2025_10_02.pdf)


### **05. 시스템 정의서(프로젝트 요약서)**

시스템 개요, 주요 기능 구조, 운영개념, 외부 연동, 고려사항 등을 요약한
프로젝트 전체 개념 정리 문서입니다.
앱이 무엇을 목표로 하고 어떤 기능들을 포함하는지 한눈에 볼 수 있습니다.

[시스템 정의서](./Byeoldori_Doc_05_SystemDef_v4_2025-11-11.pdf)

### **06. 빌드 개발 계획서**

1차~7차 빌드의 개발 목표, 일정, 구현 범위 등을 정리한
개발 진행 로드맵 문서입니다.
각 빌드에서 어떤 기능을 구현하는지 단계별로 정의합니다.

- 1차 빌드 – 관측지 정보 제공 기능(FR-400)

- 2차 빌드 – 별지도(SkyMap) 기능(FR-300)

- 3차 빌드 – 인증/회원가입/로그인 등 계정 기능(FR-600 일부)

- 4차 빌드 – 마이페이지·계정 관리 기능(FR-600 완성)

- 5차 빌드 – 관측 커뮤니티 기능(FR-500)

- 6차 빌드 – 마스코트 캐릭터(Live2D) 기능(FR-200)

- 7차 빌드 – 천체관측 교육 프로그램 기능(FR-100)

[빌드 개발 계획서](./Byeoldori_Doc_06_BuildDevPlan_2025-11-12.pdf)
